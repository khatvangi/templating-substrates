# Dispatch F4: Mode 2 simulation — code-encoded cross-descriptor templating (Test F4)

## Working directory
`/storage/kiran-stuff/templating_framework/`

## Goal
Mode 2 (code-encoded cross-descriptor templating, the genetic code's
translation mechanism) has been treated throughout v1, v2, and v3
as "Mode 1 plus a lookup table" without its own simulation. F4 builds
a Mode 2 simulation, characterizes its information capacity, tests the
framework's claim that Mode 2 inherits Mode 1's copyability, and
generates predictions for code-expansion dynamics.

The conceptual stakes: the framework's claim is that Mode 2's
information content scales as I_struct = L_X (where L_X is the input
sequence length, and Y is the output peptide), and Mode 2 is copyable
*via Mode 1 operating on the gene encoding the code*. F4 tests whether
this nested-Mode-1 copyability claim is operationally well-defined
and whether Mode 2 has properties Mode 1 alone does not.

   bounded by the code's effective alphabet size (20 amino acids,
   not 64 codons), so I_peptide < I_DNA always.

## What Mode 2 is, operationally

Mode 2 has three parts:
- A 1D sequence template X over alphabet A_X (e.g., DNA: 4 nucleotides
  in codons of length 3, so effective alphabet 64 codons).
- A separable lookup table T mapping codons to output alphabet A_Y
  (e.g., 20 amino acids + stop). The lookup is many-to-one (degenerate).
- A product Y over alphabet A_Y, generated by per-codon application of
  T to X.

Critical distinction from Mode 1: Mode 1's product alphabet equals
template alphabet (DNA → DNA via WC). Mode 2's product alphabet
differs from template alphabet, mediated by the lookup. The lookup
itself is encoded somewhere — for biology, in the tRNA charging
machinery — and that encoding is what makes Mode 2 a separate substrate.

## Pre-registered predictions

Document in script header.

P_F4_1 (information bound): For a random-codon DNA template of length
        L codons, I_struct(X→Y) where Y is the peptide is bounded by
        L × log2(|effective amino acid alphabet|). For the standard
        genetic code, I_struct ≤ L × log2(20) ≈ 4.32 L bits, less
        than Mode 1's L × log2(4) × 3 = 6 L bits per L codons. Code
        degeneracy *reduces* information capacity from the substrate
        ceiling.

P_F4_2 (degeneracy fingerprint): The product distribution P(Y|X) for
        Mode 2 with the standard genetic code shows the canonical
        degeneracy pattern — leucine is over-represented relative to
        random because 6 codons map to it; methionine is rare because
        only 1 codon maps to it. The marginal P(Y) under random X is
        the codon-count distribution, NOT uniform over amino acids.

P_F4_3 (apparatus joint signature): The G2 dual-observable apparatus
        applied to Mode 2 gives:
        - I_struct^pop scales linearly with L (matching Mode 1)
        - I_struct^chan against an ensemble {Mode 1 DNA, Mode 2
          standard code, Mode 2 alternative code, AbiK uniform}
          separates Mode 2 standard code from Mode 1 DNA via the
          product alphabet alone
        - per-base mechanism fidelity f̄ ≈ 1 − ε_translation (chosen
          per realistic translation error rate, ~10⁻⁴)

P_F4_4 (copyability via nested Mode 1): If the lookup table T is
        encoded in a separate Mode 1 substrate (a meta-genome
        encoding tRNA-aminoacyl-synthetase pairs), the Mode 2 system
        is copyable iff that meta-genome is. F4 simulates this as a
        two-level system: the gene level (Mode 1 over DNA) encodes
        the code; the protein level (Mode 2 output) is determined by
        the code applied to a separate Mode 1 substrate. The
        framework's claim is that selection acts at both levels and
        the nested system supports unbounded inheritance.

P_F4_5 (code expansion dynamics): An expanded code (24 amino acids
        instead of 20, e.g., adding pyrrolysine and selenocysteine
        plus two synthetic amino acids) increases I_struct by the
        log of the alphabet ratio: I_expanded/I_standard =
        log2(24)/log2(20) ≈ 1.061. The framework predicts code
        expansion *only* increases information capacity through Mode 1
        operating on the gene encoding the new charging machinery —
        i.e., the expansion's heritability is gated by the underlying
        Mode 1 substrate.

The strongest tests are P_F4_1 and P_F4_3. P_F4_1 sharpens the
information-content claim; P_F4_3 confirms the apparatus extends
cleanly to cross-descriptor templating.

## Concrete steps

### Step 1: Mode 2 base simulation

Implement Mode 2 with:
- DNA template X of L codons (3L nucleotides) drawn uniformly random.
- Standard genetic code (NCBI table 1).
- Per-codon translation with error rate ε = 1e-4 (canonical
  translation fidelity).
- Output Y: peptide of length L over 20 amino acids + stop.

For L ∈ {4, 8, 16, 32, 64, 128, 256, 500} codons, run 1000 reps. Compute:
- Empirical P(Y|X)
- I_struct^pop using G2's population MI definition
- Marginal P(Y) (the codon-count distribution)
- Empirical degeneracy fingerprint (relative amino acid frequencies)

Output: `results/test_f4_mode2_base_v1.csv`.

### Step 2: Mode 1 vs Mode 2 information comparison

For matched effective alphabet entropy, compare:
- Mode 1: 4-letter DNA template at length L_DNA = 3L
- Mode 2: 64-codon DNA template (encoded as 4-letter triplets) at
  length L codons, output peptide of length L

Mode 1's I_struct = 3L × log2(4) = 6L bits. Mode 2's I_struct
should be ≤ L × log2(20) ≈ 4.32L bits, less than Mode 1's
substrate-level information by a factor of 6/4.32 ≈ 1.39.

Output: `results/test_f4_mode1_vs_mode2_v1.csv`.

This confirms (or falsifies) P_F4_1.

### Step 3: G2 apparatus signature for Mode 2

Compute the joint apparatus signature (I_struct^pop, I_struct^chan, f̄)
for Mode 2 standard code at L=64, against the channel ensemble
{Mode 1 DNA, Mode 2 standard, Mode 2 swapped (random permutation of
codon→aa map), AbiK uniform}.

Predicted:
- I_struct^pop ≈ 4.32 × 64 = 277 bits (population entropy of the
  peptide ensemble at L=64)
- I_struct^chan: separation from Mode 1 DNA is high (different
  alphabets); separation from Mode 2 swapped is moderate (same
  alphabet, different code mapping); separation from AbiK uniform
  is high (Mode 2 has structured marginal P(Y))
- f̄ ≈ 0.9999 (ε = 1e-4)

Output: `results/test_f4_apparatus_signature_v1.csv`.

This confirms (or refines) P_F4_3.

### Step 4: Nested-Mode-1 copyability simulation

Two-level system:
- Meta-level: Mode 1 substrate (gene) encoding a 64-entry codon→aa
  lookup table. 64 codons × 5 bits per amino-acid index = 320 bits
  per gene. Mutation rate μ_meta = 1e-4 per bit.
- Phenotype level: a fixed-distribution X is translated through the
  meta-level's lookup to produce Y.

Population dynamics: K = 200 agents, each carrying a (gene, X→Y
realization) tuple. Selection on phenotype Y toward a fixed target
peptide. Reproduction copies the gene with mutation, re-runs translation.

For 500 generations, track: gene diversity, phenotype fitness, code
fidelity (how well the lineage's code converges to a fitness-
maximizing code).

Output: `results/test_f4_nested_copyability_v1.csv`.

The framework's prediction: the population evolves toward a
fitness-favorable code, and that code propagates by Mode 1 inheritance
of the gene. Selection acts on phenotype but the heritable substrate
is the gene encoding the code. This is the operational test of
P_F4_4.

### Step 5: Code expansion test

Compare standard 20-aa code vs expanded 24-aa code at L=64. For each:
- Generate 1000 random DNA templates, translate, compute I_struct^pop
- Compute the marginal entropy H(Y) — this is the population
  information content of the peptide product

Predicted H(Y) for standard code: ~3.59 bits per position (entropy
of codon-count distribution over 20 aa).

Predicted H(Y) for 24-aa expanded code: ~3.79 bits per position.

Ratio: 1.056. Test whether the empirical ratio matches 1.056 ± noise.

Output: `results/test_f4_code_expansion_v1.csv`.

### Step 6: v3 statement

Generate `results/test_f4_v3_statement.md` containing:
1. Mode 2's empirical information bound at multiple L values.
2. The Mode 1 vs Mode 2 capacity comparison.
3. The apparatus signature for Mode 2.
4. The nested-copyability simulation outcome — does selection on

5. The code expansion result — does I_struct scale as predicted?
6. A drop-in paragraph for v3 Results section "Mode 2 (translation):"

   Mode 1) but inherits Mode 1's copyability through the gene level.

Without this dispatch, v3's Mode 2 paragraph is descriptive. With
this dispatch, v3 has empirical numbers and a falsifiable claim about
code expansion dynamics.
